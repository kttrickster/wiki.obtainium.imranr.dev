# Обзор интерфейса

Нижняя панель является основным способом навигации по Obtainium.

## Страница «Приложения»

Это главный экран; он содержит список приложений, отслеживаемых Obtainium, и предоставляет основную информацию для каждого приложения.

На этой странице вы можете использовать кнопки в нижней части экрана для выполнения различных операций с несколькими приложениями одновременно. Это можно сделать после выбора одного или нескольких приложений, либо с помощью долгого нажатия для перехода в режим множественного выбора, либо с помощью кнопки "выбрать все". Доступные пакетные операции включают обновление, установку, удаление, категоризацию, закрепление/открепление, обмен URL-адресами приложений или ссылками экспорта, а также пометку выбранных приложений как обновлённых.

Приложения можно отнести к категориям, которые отображаются в виде цветных полос по левому краю каждой строки приложения. Если в настройках включено "Группировать по категориям", то список приложений делится на разворачиваемые разделы по категориям.

На экране представления информации отдельного приложения можно просматривать информацию о версии, дату выпуска, журнал изменений и хеши сертификатов подписи APK (удерживайте чтобы скопировать). Если обнаружено переименование репозитория, появится заметное уведомление с возможностью автоматической замены URL-адреса приложения.

Вы также можете отфильтровать определённые приложения, нажав кнопку со значком лупы. Это позволит вам просматривать только те приложения, которые соответствуют заданным критериям (например, установленные/неустановленные, наличие обновлений, источник, категория и т.д.).

## Страница «Добавить»

Эта страница позволяет добавить приложение по его URL-адресу источника.

При вводе URL-адреса источника Obtainium автоматически определит, какую [логику источника](app_tracking.md/#_2) он должен использовать, и представит соответствующие параметры. Если URL не соответствует ни одному из поддерживаемых источников, по умолчанию будет выбран источник "[HTML](sources.md/#html)". Этот источник является общим запасным вариантом, который может сработать в некоторых случаях. Если вы считаете, что Obtainium сделал неправильный выбор, вы можете вручную указать тип используемого источника.

Для большинства источников вам не нужно предоставлять точный URL-адрес; например, любой URL-адрес GitHub, который содержит название репозитория и его владельца, будет приниматься источником GitHub (например, `https://github.com/ImranR98/Obtainium/releases/latest` будет автоматически сокращён до `https://github.com/ImranR98/Obtainium`). Однако ваш URL-адрес должен быть более точным в двух ситуациях:

1. При использовании источника HTML
    - Например, APK-файл Tor находится по адресу `https://www.torproject.org/download/`, поэтому ввод только `https://www.torproject.org/` не будет работать.
2. При ручном выборе источника (переопределение выбранного Obtainium запасного варианта HTML)

Эта страница также позволяет вам искать приложения из источников, которые поддерживают эту функцию (страница [Данные](#_4) также предоставляет отдельный инструмент поиска). Обратите внимание, что если приложение отображается в результатах поиска, это не означает, что оно будет успешно добавлено - ему всё ещё необходимо выполнить все остальные критерии.

Наконец, на этой странице перечислены все поддерживаемые источники, а также дополнительная информация, например, доступен ли поиск.

## Страница «Данные»

На этой странице вы можете экспортировать данные Obtainium, чтобы впоследствии их можно было импортировать. Вы также можете включить автоматический экспорт, который будет происходить при каждом изменении данных, и контролировать, будут ли приватные данные (такие как API-токены) включены в экспорт.

На этой странице также представлены другие способы импорта большого количества приложений:

- **Поиск**: Поиск среди поддерживаемых источников и импорт сразу нескольких результатов.
- **Импорт из списка URL-адресов**: Вставить или загрузить файл со списком URL-адресов приложений, чтобы импортировать их все сразу.
- **Массовый импорт из источника**: Импортировать приложения из источников с подборками URL, такие как избранные репозитории пользователя на GitHub.

Инструмент поиска на этой странице также позволяет искать в источниках, которые требуют указать хост/домен, такие как [сторонние репозитории F-Droid](sources.md/#f-droid).

## Страница «Настройки»

На этой странице представлены различные настройки интерфейса и поведения, организованные в разделы:

- **Обновления**: Интервал проверки обновлений в фоновом режиме, приоритетная служба для проверки обновлений, ограничение при отсутствии Wi-Fi/не на зарядке, проверка наличия обновлений при запуске, параллельные загрузки, интеграция Shizuku для тихих установок (с возможностью притвориться Google Play), и интеграция AppVerifier для проверки APK перед установкой.
- **Настройки источников**: Данные для определённых источников (например, токены GitHub/GitLab) и другие настройки конфигурации для конкретных источников.
- **Внешний вид**: Тема (как в системе/светлая/тёмная), чёрная тема, динамические цвета Material You, пользовательский цвет акцента, сортировка, язык, системный шрифт, просмотр веб-страниц внутри приложения, группировка по категориям и настройки перехода по страницам.
- **Категории**: Создавайте, переименовывайте и удаляйте категории для организации ваших приложений.

В конце страницы настроек также содержатся ссылки на исходный код, эту вики, коллективно собранные конфигурации приложений и журналы приложения для устранения неполадок.
